# Spark

  • 一个基于内存计算的开源的集群（分布式）计算系统
  • 效率高于 Hadoop 。Job 中间输出的结果均保存在内存中，从而不再需要读写 HDFS，节省了磁盘IO 耗时
  • 性能比 Hadoop 快约 100 倍

# Scala 的条件表达式

  • if/else 语法结构和 Java、C++ 一样
  • Scala 中，if/else 是表达式，有值，这个值就是跟在 if 或 else 之后的表达式的值

# Scala 循环

  • Scala 拥有与 Java 和 C++ 相同的 while 和 do 循环
  • Scala 中，可以使用 for 和 foreach 进行迭代

# Scala 函数参数的求值策略

  • Scala 中，有两种函数参数的求值策略
    • Call By Value：对函数实参求值，且仅求一次
    • Call By Name：函数实参每次在函数体内被用到时都会求值
  • Scala 通常使用 Call By Value
  • 如果函数的形参类型以 => 开头，那么就会使用 Call By Name
def foo(x:Int) = x    // Call By Value
def foo(x: => Int) = x    // Call By Name

# Scala 的 Lazy 值

当 Scala 中的值被申明为 lazy 时，它的初始化将被推迟，直到我们首次对它取值

# Scala 函数式编程

在 Scala 中，函数和数字一样，可以在变量中存放函数：

import scala.math._
val num = 3.14
val fun = ceil._

即是：将函数作为变量的值

# 匿名函数和带函数参数的函数

  • 匿名函数
    • 在 Scala 中，不需要给每一个函数命名，正如不用给每个数字命名一样。
  • 带函数参数的函数
    • 在 Scala 中，可以将一个函数作为另外一个函数的参数。

# 闭包

类似于函数的嵌套，在一个函数定义中，包含另外一个函数的定义

# 柯里化 Currrying

柯里化函数（Currying Function）是把具有多个参数的函数转换成一条函数链，每个节点上是单一参数

以下两个 add 函数定义是等价的
def add(x:Int, y:Int) = x + y
def add(x:Int)(y:Int) = x + y // Scala柯里化的语法

# 集合

  • 可变集合
  • 不可变集合
    • 集合从不改变，因此可以安全地共享其引用。

  • 不可变列表（List）
  • 可变列表（LinkedList）

# 序列

常用的序列：Vector、Range

  • Vector：是 ArrayBuffer 的不可变版本，是一个带下标的序列
  • Range：表示一个整数序列

# 集和集的操作

  • 集 Set 是不重复元素的集合
  • 集和列表不同，它不保证元素插入的顺序。默认以 Hash 集实现

# 模式匹配

Scala 有一个强大的模式匹配机制，可以应用在很多场合：

  • switch 语句
  • 类型检查
  • Scala的守卫
  • 模式匹配中的变量
  • 匹配数组和列表

Scala 还提供了样本类（case class），对模式匹配进行了优化

# 什么是 Case Class

Scala 的 case class 就是在普通的类定义前加 case 这个关键字，然后你可以对这些类来进行模式匹配

case class 带来的最大的好处是它们支持模式识别
