# try、catch、finally 用法总结

# try、catch、finally 执行顺序

try 和 catch 逻辑基本相同，try 块中出现异常跳转到 catch，若 catch 中出现异常则跳转到 finally。

try 或 catch 正常执行，若存在 return，则先执行 return 的代码并保存返回值信息（基本类型保存值信息，引用类型保存地址信息），然后执行 finally，若 finally 中出现异常或包含 return，则执行结束。若是没有异常且没有 return，则执行 try 或 catch 中的 return结束。

# 没有 return

当 try-catch-finally 中无 return 时，如果 try 块中无异常时执行完 try 块直接执行 finally 块。如果 try 块中出现异常则进入 catch 中，无论 catch 中是否出现异常都会执行 finally 代码块。finally 块执行结束后，如果 catch 块有出现异常，那么抛出 catch 块的异常，否则的话直接跳出。

下面以 catch 中抛出异常为例子，可以发现 catch 抛出异常后还是会执行 finally，在 finally 执行完之后抛出关于 catch 的异常。

public static int testNoReturn(){
        int res = 1;
        try{
            res++;
            System.out.println("try ======== res:"+res);
            int a = 1/0;
        }catch (Exception e){
            res++;
            System.out.println("catch ======== res:"+res);
            int a = 1/0;
        }finally {
            res++;
            System.out.println("finally ======== res:"+res);
        }
        return res;
    }

# 存在 return

try 或 catch 中存在 return 时流程是基本一致的。但是当finally 中包含 return 时会覆盖 try 或 catch 中的 return 值，而且会覆盖 catch 中抛出的异常信息。（但是此时try cache 中 return 的是 基本数据类型 或者是 引用类型这里有区别）

try 或 catch 中存在 return 的话将返回值进行保存（基本数据类型直接保存，引用数据类型的话保存引用地址），然后去执行 finally 块中的代码，finally 块中的代码执行结束后直接返回（这里的直接返回是在 finally 没有 return 的情况下可以直接返回。如果 finally 中有 return 的话，这里是会覆盖掉 try 或 catch 中返回的 return）。

因为引用类型返回时保存的是地址，所以如果在 finally 中进行修改的话，是会对返回的结果造成影响的。但是如果此时赋值是为 null 的话，信息并未发生变化，所以此时返回值依然是地址指向的对象。

但是当 finally 中存在 return 的话会覆盖 try 或 catch 中返回的数据。如果 catch 中抛出异常时会将异常覆盖掉（无论是基本数据类型或者是引用数据类型都是一样的）。

# 总结

  1. 无 return 且 没有出现异常的话，try -> finally；出现异常的话 try -> catch -> finally 。
  2. try 或 catch 有return 的话，如果 return 的是基本的数据类型，finally 代码执行完是不会发生更改的（建立在 finally 中没有 return 的情况下）；如果 return 的是引用数据类型，finally 代码执行完是会对返回数据产生影响的。
  3. finally 中如果存在 return，会覆盖掉 try 或者 catch 中的返回值信息。若是 try 或 catch 中抛出异常也会被 finally 中的 return 覆盖掉。

# Java 基本类型和包装类型的区别

  • 包装类型不赋值就是null，基本类型有默认值
  • 包装类型可用于泛型，基本类型不可以
  • 包装类型属于对象类型，几乎所有的对象实例都存在于堆中。基本类型的局部变量存放在 Java 虚拟机栈的局部变量表中，成员变量存放在 Java 虚拟机的堆中。
  • 相较于包装类型，基本数据类型的占用空间较小

# 自动装箱与拆箱

  • 自动装箱：将基本类型用相对应的引用类型包装起来
  • 拆箱：将包装类型转换成基本数据类型

# 面向对象和面向过程的区别

两者的区别在于解决问题的方式不同：

  • 面向过程把解决问题的过程拆成一个个方法，通过方法来解决问题

面向过程分析出解决问题的步骤，然后函数把这些步骤一步一步执行，使用过的时候一个一个依次调用就可以了。

  • 面向对象会先抽象出对象，然后用对象执行的方法来解决问题

面向对象就是把构成问题事务分解成各个对象，简历对象的目的是为了描述某个事务在解决问题的时候的步骤中一步步的行为

# 接口和抽象类有什么共同点和区别？

共同点

  • 不能被实例化
  • 都可以包含抽象方法
  • 都有默认的实现方法

区别

  • 定义的关键字不同，接口使用 interface 定义，抽象类使用 abstract 定义；
  • 接口使用 implements 关键字定义具体实现，抽象类使用 extends 进行继承
  • 一个类只能继承一个抽象类，但是可以实现多个接口

  • 接口中属性的访问控制符只能是 public，抽象类中的属性控制符可以是无限制的

  • 接口用于对类的行为进行约束，实现了某个接口就具有了对应的行为。抽象类主要用于代码复用，强调的是所属的关系
  • 接口中的方法默认控制符只能是 public，抽象类中的方法控制符无限制（但是抽象方法不能使用 private 修饰0

  • 接口中的成员变量只能是 public static final 类型，不能被修改且必须有初始值；抽象类的成员变量默认 default，可在子类中被重新定义，也可以重新赋值
  • 接口中的普通方法不能有具体的实现，但是 JDK 1.8 之后，static 和 default 方法必须有方法实现；抽象类中的普通方法必须有方法实现，抽象方法不能有方法实现

  • 接口中不能使用静态代码块；抽象类中可以使用静态代码块

# 深拷贝和浅拷贝有什么区别？什么是引用拷贝？

  • 浅拷贝：浅拷贝会在堆上创建一个新的对象，不过，如果原对象内部的属性是引用类型的话，浅拷贝会直接复制内部对象的引用地址，也就是说拷贝的对象和原对象使用的是同一个对象；

  • 深拷贝：深拷贝会完全复制整个对象，包括这个对象所包含的内部对象；

  • 引用拷贝：引用拷贝就是两个不同的引用指向同一个对象；和浅拷贝的区别就是引用拷贝是直接指向的地址，然后浅拷贝的话是有在堆上复制出来一个对象的；

# == 和 equals() 有什么区别？

== 对于基本类型变量和引用类型变量的效果是不同的

  • 对于基本来说来说，== 比较的是值
  • 对于引用类型来说，== 比较的是对象的内存地址

因为 Java 只有值传递，所以 == 本质上来说都是比较的值是否相等

equals() 不能用来判断基本数据类型的变量，只能用来判断两个对象是否相等。equals() 方法存在 Object() 类中，而 Object() 类是所有类的父类，所以所有的类都有 equals() 方法。

equals() 存在两种情况；

  • 没有重写方法：通过 equals() 比较时，等价于 == 比较两个对象，使用的是默认的 equals() 方法
  • 重写方法：一般都通过重写方法来比较类中的属性，属性相等，返回 true；

public boolean equals(Object anObject) {
    if (this == anObject) {
        return true;
    }
    if (anObject instanceof String) {
        String anotherString = (String)anObject;
        int n = value.length;
        if (n == anotherString.value.length) {
            char v1[] = value;
            char v2[] = anotherString.value;
            int i = 0;
            while (n-- != 0) {
                if (v1[i] != v2[i])
                    return false;
                i++;
            }
            return true;
        }
    }
    return false;
}
  • 首先判断对象是否属于同一个类型
  • 是的话强转换同类型，并判断长度是否相等
  • 如果相等的话，判断每一个字符是否都相等
  • 如果不相等就返回 false
  • 如果都相等就返回 true

# String、StringBuilder、StringBuffer区别？

可变性

  • String 是不可变的
  • Stringbuilder 与 StringBuffer 都是继承自 AbstractStringBuilder 类，在 AbstractStringBuilder 中也是使用字符串数组保存字符串，不过没有 final 和 private 关键字修饰，最关键的是 AbstractStringBuilder 类还提供了好呢多修改字符串的方法，如 append、insert、indexOf 等方法。

线程安全性

  • string 中的对象是不可变的，可以理解为常量，线程安全；
  • StringBuffer 对方法加了同步锁或者对调用的锁加了同步锁（synchronized），线程安全；
  • StringBuilfer 没有对方法进行添加同步锁操作，线程不安全；

性能

  • String 每次进行改变的时候，都会重新生成一个新的 String 对象，然后将对象指向新的 String 对象；
  • StringBuffer 会对本身进行操作，而不是生成新的对象并改变对象引用（适合多线程操作字符串）；
  • StringBuilder 相比较 StringBuffer 会获得 10% ~ 15% 的性能提升，但是要承担线程不安全的风险（适合但单线程操作字符串）；

# 序列化与反序列化

序列化：序列化是将对象的状态信息转换成可以存储或者传输的形式的过程。在序列化期间，对象将当前状态写入到临时或者持久性存储区。反序列化的时候而已从存储区中读去序列化对象的状态，重新创建该对象。

序列化主要有两个用途：

  • 把对象的字节序列永久保存到硬盘上，通常存放在一个文件中（序列化对象文件）
  • 在网络上传输对象的字节序列（网络传输对象）

实际上就是将数据持久化，防止数据一致存储在内存中，消耗内存资源，而且序列化后的对象数据也能更好地进行网络传输

# 如何对对象进行序列化呢？

Java 中目前还没有一个关键字是直接定义一个所谓的 ”持久化“ 对象，对象的持久化和反持久化都是需要程序员在代码里手动显式地进行序列化和反序列化还原的动作。

# 一、你能回答正确吗

如上定义的为常量；

String b = a +"456";

如上 b 为变量，为啥？ 难道常量拼接常量得到的是变量吗？

不是，常量拼接常量得到的依旧是常量。

# 二、String定义常量和变量的区别

我们再来通过两种写法分析：

String a = "123";
String b = new String("123");

如上第1行，定义了一个常量 a ，第2行，通过关键字 new 的形式，创建了一个变量 b 。

我们结合之前学过的 JVM 再深入一些，第1行在常量池开辟了一块空间，存放字符串 123，通过 a 对象指向这个常量对象。第2行由于使用了 new 关键字，所以会在堆空间中开辟一块内存区域，在其中存放字符串 123，并把内存的地址赋予 b 变量。

所以， a==b 吗？显示是 false，一个是堆内存，一个是常量池。

如果将 a 修改成：

String a = new String("123")；

那么，a==b 吗？

依旧是 false。

为什么？只要通过 new 形式，自然是创建两个对象。即使是他们的值是一致的，也是 false。

# 三、String常量变量的总结

String 常量存放在常量池中，JVM 处于优化考虑，会让内容一致的对象共享内存块，但变量是放在堆空间中的，new 定义的不同变量内存地址不同。

String 常量连接常量，还是常量，依旧用常量池管理，但常量连接变量就是变量了。

# 四、创建几个对象的练习

以下几种情况(均不考虑字符串在常量池中已存在的情况)：

1、String a="123";

创建了1个对象

jvm在编译阶段会判断常量池中是否有 "123" 这个常量对象如果有，a直接指向这个常量的引用，如果没有会在常量池里创建这个常量对象。

2、String a=new String("123");

创建了2个对象

同情况1，jvm编译阶段判断常量池中 "123"存在与否，进而来判断是否创建常量对象，然后运行阶段通过new关键字在java heap创建String对象。

3、String a="123"+"456";

创建了1个对象

jvm编译阶段过编译器优化后会把字符串常量直接合并成"123456"，所有创建对象时最多会在常量池中创建1个对象。

4、String a="123"+new String("456");

创建了4个对象

常量池对象"123" ,"456"，new String("456")创建堆对象，还有一个堆对象"123456"。

# ThreadLocal原理

可以看到，ThreadLocalMap 中的 key 是 ThreadLocal，而 ThreadLocalMap 又是 Thread 中的成员变量，属于每一个 Thread 实例对象。

所以，ThreadLocal 不会拷贝共享变量，它解决并发问题的原理很简单，就是要求开发者为每一个线程 “发布” 一个变量，然后这个变量本身就是线程隔离的。

ThreadLocal 的作用，主要是跳过方法的参数列表在线程内传递参数。

# ThreadLocal 的内存泄漏

从 ThreadLocalMap 的源码可以看出，Entry 继承自 WeakReference，并且会将 ThreadLocal 添加到弱引用队列中。

我们知道，弱引用对象只能存活到下一次GC。所以说如果 ThreadLocal 没有关联任何强引用，只有 Entry 上的弱引用的话，发生一次 GC 后 ThreadLocal 就会被回收，就会出现 ThreadLocalMap 上面关联 Entry，但是 Entry 上面没有 key 的状态。但是此时 value 仍然关联在 ThreadLocalMap 上，只是说没有办法通过常规手段获得 value，从而会造成内存泄漏（虽然说线程销毁后会释放内存，但是在线程执行期间，始终都是存在这么一种内存占用的情况）。

# 如何避免内存泄漏？

为了避免出现内存泄漏的情况，Java 建议设置静态的 ThreadLocal 变量，保证其一直存在与之关联的强引用关系。

ThreadLocal 自身也做了一些努力去清除这些没有 key 的 Entry：

最重要的还是开发者主动调用 remove() 方法清除无用的变量才是正确使用 ThreadLocal 的方式。

package com.BXuan.LeetCode;

import java.util.HashMap;
import java.util.Map;

public class LRUCacheDemo02 {
    // 定义节点信息
    static class Node<K, V>{
        K key;
        V value;
        Node<K, V> pre;
        Node<K, V> next;
        public Node(){
            // 这里是对初定义的头节点和尾节点进行初始化
            this.pre = this.next = null;
        }
        public Node(K key, V value){
            this.key = key;
            this.value = value;
            // 这里是对初定义的头节点和尾节点进行初始化
            this.pre = this.next = null;
        }
    }
    // 定义双向链表
    static class DoubleLinkedList<K, V>{
        Node<K, V> head;
        Node<K, V> tail;
        // 对双向链表进行初始化
        public DoubleLinkedList(){
            head = new Node<>();
            tail = new Node<>();
            head.next = tail;
            tail.pre = head;
        }
        // 添加头节点操作
        public void addHead(Node<K, V> node){
            node.next = head.next;
            node.pre = head;
            head.next.pre = node;
            head.next = node;
        }
        // 删除节点操作
        public void remove(Node<K, V> node){
            node.pre.next = node.next;
            node.next.pre = node.pre;
            node.pre = null;
            node.next = null;
        }
        // 获得最后一个节点
        public Node<K, V> getLast(){
            return tail.pre;
        }
    }
    // 初始化容量
    private final int cacheSize;
    // 定义map用于查询
    Map<Integer, Node<Integer, Integer>> map;
    // 定义双向链表用于查询和删除
    DoubleLinkedList<Integer, Integer> doubleLinkedList;
    //初始化
    public LRUCacheDemo02(int cacheSize){
        this.cacheSize = cacheSize;
        map = new HashMap<>();
        doubleLinkedList = new DoubleLinkedList<>();
    }
    // 得到某个关键值操作
    public int get(int key){
        if (!map.containsKey(key)){
            return -1;
        }
        Node<Integer, Integer> node = map.get(key);
        doubleLinkedList.remove(node);
        doubleLinkedList.addHead(node);
        return node.value;
    }
    // 将关键值放回操作
    public void put(int key, int value){
        if (map.containsKey(key)){
            // 更新操作
            Node<Integer, Integer> node = map.get(key);
            node.value = value;
            map.put(key, node);

            doubleLinkedList.remove(node);
            doubleLinkedList.addHead(node);
        }else {
            if (map.size() == cacheSize) {
                Node<Integer, Integer> lastNode = doubleLinkedList.getLast();
                map.remove(lastNode.key);
                doubleLinkedList.remove(lastNode);
            }
            Node<Integer, Integer> newNode = new Node<>(key, value);
            map.put(key, newNode);
            doubleLinkedList.addHead(newNode);
        }

    }

    public static void main(String[] args) {
        LRUCacheDemo02 lruCacheDemo = new LRUCacheDemo02(3);
        lruCacheDemo.put(1, 1);
        lruCacheDemo.put(2, 2);
        lruCacheDemo.put(3, 3);
        System.out.println(lruCacheDemo.map.keySet());
        lruCacheDemo.put(4, 4);
        System.out.println(lruCacheDemo.map.keySet());
        lruCacheDemo.get(2);
        lruCacheDemo.put(1, 0);
        System.out.println(lruCacheDemo.map.keySet());
    }
}

# 1、Lambda表达式

Lambda是一个匿名函数，可以将表达式理解为一段可以传递的代码

Lambda表达式（也称“语法糖”）的基础语法：Java8中引入了一个新的操作符："->"，该操作符称为箭头操作符或者Lambda操作符，箭头操作符将Lambda表达式拆分成两部分

左侧：Lambda表达式的参数列表

右侧：Lambda表达式中所需执行的功能，即Lambda体

下面是其语法格式：

  • 语法格式1：无参数无返回值
Runnable runnable2 = () -> System.out.println("我是用Lambda表达式实现的HelloWorld!");
  • 语法格式2：有参数无返回值
Consumer<String> consumer = (x) -> System.out.println(x);
  • 语法格式3：有一个参数无返回值，小括号可以省略不写
Consumer<String> consumer = x -> System.out.println(x);
  • 语法格式4：有两个以上的参数并且Lambda体中有多条语句
Comparator<Integer> comparator = (x, y) -> {
    System.out.println("函数式接口");
    return Integer.compare(x, y);
};
  • 语法格式5：若Lambda体中只有一条语句，return和大括号都可以省略不写
Comparator<Integer> comparator = (x, y) -> Integer.compare(x, y);
  • 语法格式6：Lambda表达式的参数列表的数据类型可以省略不写，因为JVM编译器通过上下文推出数据类型，

即“类型判断”

Comparator<Integer> comparator = (Integer x, Integer y) -> Integer.compare(x, y);

口诀：

左右遇一括号省

左侧推断类型省

# 二、函数式接口

# Java8 内置四大核心函数

Consumer<T>：消费型接口：void accept(T t)
Supplier<T>：供给型接口：T get()
Function<T, R>：函数型接口：R apply(T t)
Predicate<T>：断言型接口：boolean test(T t)

public class TestLambda03 {
    // consumer<T> 消费型接口
    @Test
    public void test01(){
        happy(10000, (m) -> System.out.println("逛街花了" + m + "元！"));
    }
    public void happy(double money, Consumer<Double> consumer){
        consumer.accept(money);
    }

    // Supplier<T> 供给型接口
    @Test
    public void test02(){
        List<Integer> result = getNumberList(10, () -> (int) (Math.random() * 100));
        System.out.println(result);
    }
    // 需求：产生指定个数的整数，并放到集合中
    public List<Integer> getNumberList(int num, Supplier<Integer> supplier){
        ArrayList<Integer> list = new ArrayList<>();
        for (int i = 0; i < num; i++) {
            Integer integer = supplier.get();
            list.add(integer);
        }
        return list;
    }

    // R Function<T>：函数型接口
    @Test
    public void test03(){
        String result = stringHandler("aaabbbccc", (x) -> x.trim().toUpperCase());
        System.out.println(result);
    }
    // 需求：用于处理字符串
    public String stringHandler(String string, Function<String, String> function){
        return function.apply(string);
    }

    // Predicate<T>：断言型接口
    @Test
    public void test04(){
        List<String> list = Arrays.asList("Hello", "BXuan", "atguigu", "www", "meidi", "GDUT");
        List<String> result = filterString(list, (x) -> x.length() > 4);
        System.out.println(result);
    }
    // 将满足条件的字符串，放入集合中
    public List<String> filterString(List<String> list, Predicate<String> predicate){
        List<String> result = new ArrayList<>();
        for (String s : list) {
            if (predicate.test(s)){
                result.add(s);
            }
        }
        return result;
    }
}

# 四、stream API

# 1、概念及操作步骤

Stream（流）：是数据渠道，用于操作数据源（集合、数组等）所生成的元素序列。

“集合讲的是数据，流讲的是在数据传输过程中发生的一系列计算”

注意：

  1. Stream 自己不会存储元素
  2. Stream 不会改变源对象。相反，会返回一个持有新结果的Stream
  3. Stream 操作是延迟执行的，即是等待需要结果的时候才会执行

操作步骤

  • 创建

一个数据源（集合、数组等），获取一个流

  • 中间操作

一个中间操作链，对数据源的数据进行操作处理

  • 终止操作（终端操作）

一个终止操作，执行中间操作链，并产生结果

public class TestStreamApi01 {
     @Test
    public void test01(){

         List<String> list = new ArrayList<>();

         Employee[] employees = new Employee[10];
         Stream<Employee> stream02 = Arrays.stream(employees);
         // 3、通过Stream类中的静态方法of()
         Stream<String> stream03 = Stream.of("aa", "bb", "cc");

public static Stream<Character> filterCharacter(String string){
    List<Character> list = new ArrayList<>();
    for (Character character : string.toCharArray()){
        list.add(character);

# 五、接口中的默认方法与静态方法

# 1、接口默认的“类优先”原则

若一个接口中定义了一个默认方法，而另外一个父类或接口中又定义了一个同名的方法时

  • 选择父类中的方法。如果一个父类提供了具体的实现，那么接口中具有相同名称和参数的默认方法会被忽略
  • 接口冲突。如果一个父接口提供一个默认方法，而另一个接口也提供了一个具有相同名称和参数列表的方法（不管方法是否为默认方法），那么必须覆盖该方法来解决冲突。

下面根据多接口和父类优先的原则，举几个例子简单感受一下；

先建立第一个接口，第一个接口中还包含一个方法，在控制类中还可以通过调用接口中的方法直接实现操作。

public interface TestInterface01 {
    default String getName(){
        return "哈哈哈";
    }
    public static void show(){
        System.out.println("接口中的静态方法");
    }
}

继续建立第二个接口：

public interface TestInterface02 {
    default String getName(){
        return "嘿嘿嘿";
    }
}

下面建立一个父类：

public class MyClass {
    public String getName(){
        return "呵呵呵";
    }
}

在这里建立一个接口实现类，继承父类，然后对接口1、接口2进行多实现的方法。

public class InterfaceImpl extends MyClass implements TestInterface01, TestInterface02{

}

下面建立一个测试类并查看结果

public class TestInterface {
    @Test
    public void test01(){
        TestInterface01.show();
    }
    @Test
    public void test02(){
        InterfaceImpl i = new InterfaceImpl();
        System.out.println(i.getName()); // 呵呵呵
    }
}

可以明显的看出：

  1. test01()方法是直接调用接口1中的show方法，该方法直接返回 接口中的静态方法。说明可以直接调用接口中已经实现了的方法
  2. test02()方法是对接口实现类进行简单的调用，此时返回的是 呵呵呵。这里是对父类的方法进行实现，秉承的也是 父类优先的原则
  3. 如果没有继承父类，但是同时实现了多个接口，那么要看接口实现类中是对哪个接口中的方法进行了实现，那么测试类中实现的就是哪个接口中的方法。
